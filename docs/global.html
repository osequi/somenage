# constant breakpoints

Defines the breakpoints for the theme.

Breakpoints are an array of {name, value} pairs.

  • name comes from TBreakpointNames.
  • value is unitless but represents pixels.

Breakpoints follow the mobile first design / progressive enhancement pattern. This implies media queries on breakpoints will use the min-width approach. See useBreakpoints.

See:
  • https://abookapart.com/products/mobile-first
  • https://www.uxpin.com/studio/blog/a-hands-on-guide-to-mobile-first-design/

Example
[{name: 'mobile', value: 320}, {name: 'tablet', value: 768}, ...]

# TLinearScale

Defines the linear scale type.

The name of the scale. It's always linear.

# TModularScale

Defines the modular scale type.

The various scale types share the same structure, like name and settings. This way they are pluggable into the theme.

Properties:
Name Type Description
name string

The name of the scale. It's always modular.
settings object

The settings for the scale. It comes from modularscale-js.
base Array

The base(s) of the scale(s)., A base defines the value of the 0th element on the scale. For example base=1 makes the 0th element size 1em., modularscale-js supports multiple bases.
ratio number

The logic of scaling. See Ratios.
